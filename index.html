http协议

1.网络模型

网络模型即互联网中指定的准则（OSI七层模型架构）或者TCP/IP协议（规定两台计算之间如何进行消息传输）。在网络模型中一直存在两种角色，分别为客户端与服务端。

1.1客户端

任何一个计算机都可以作为一个客户端使用

1.2服务端

提供一种功能性的服务计算。例如提供数据存储的服务、提供一个浏览器的访问功能。

客户端与服务端没有明确的界限。一般每一台计算机即可作为客户端使用也可以作为服务端使用。

1.3客户端与服务端的通信!

  1. 客户端以TCP/IP协议的方式与服务器建立连接
  2. 客户端发送请求request（发送数据）给服务器
  3. 服务器响应请求response
  4. 客户端断开连接

2.网站的访问原理

在http协议中提供的一种叫做web服务

2.1请求中的URL

通过URL（统一资源定位让服务端寻找某一个文件提供服务）

完整的URL格式

协议：//域名：/端口/虚拟目录的文件地址

http：//www.baidu.com/imges/longin.png

2.2单个求情的访问原理

  1. 客户端（浏览器），输入要访问的地址执行
  2. 本地的DNS解析生效获得与域名相对应的IP地址。如果可以获取到，直接使用，否则使用第3步
  3. 请求在整个互联网中DNS解析的服务器。绝对可以得到IP地址
  4. 客户端借助于TCP/IP协议与服务端建立连接
  5. 客户端按照HTTP协议格式将数据包发送给服务端
  6. 服务端接受到请求所发送的数据在进行相关的运算
  7. 服务端再按照HTTP协议格式将数据发送给客户端
  8. 最终客户端获取到的结果按照固定的格式显示结果给用户
  9. 断开连接

2.3单个页面的请求流程

在一个网页的访问中会触发多次请求。具体一个网页的请求次数与网页中所使用的资源个数存在关系。

  1. 客户端发起某一个地址的请求（http://www.baidu.com）
  2. 服务端返回对应的HTML结构
  3. 客户端解析HTML结果显示在浏览器中
  4. 客户端会根据HTML中使用的资源地址（图片、样式文件、js文件）再次逐个发送请求

不论在浏览器如何访问某一个地址，一定在服务器上存在一个对应的文件。看不到只是被隐藏了。

3.什么是http协议

  • 超文本传输协议（按照一定的格式传输’HTML‘）Hyper Text Transfer Protocol
  • 超文本传输协议（http）是一种通讯协议，它允许将超文本标记语言（HTML）文档从web服务器传到Web浏览器。
  • HTTP工作在TCP/IP 协议体系中的协议上
  • http协议属于无状态的协议，连续两次请求没有相互的联系（目前http协议存在长链接方式保持连接。

5.http协议

5.1请求方式

按照web服务，根据URL地址提供服务。服务过程中使用http协议固定格式。在格式中存在请求方式的约束。在http协议中规定了八种请求方式，每一种请求方式具备了不同的含义。

常用方式：

  • get请求：get请求表示要获取到数据，例如浏览器中直接输入地址访问
  • post请求：post请求表示要添加数据，常见在form表单中提交数据，将method属性设为post
  • put请求：put请求表示要修改数据
  • delete请求：delete请求表示要删除数据

虽然请求方式规定每一种请求完成某种数据操作，但是并不是绝对，可以使用get请求完成数据的删除 使用post请求完成数据的修改

get和post请求的区别：

  1. get请求参数在URL地址中
  2. post请求参数存在请求体中
  3. get请求所带的参数大小有限

5.2http请求

http协议的请求区分四个部分，分别为请求行、请求头、空白行、请求体

关于http协议请求的格式说明：

①请求行：

​ 请求行中有请求方式、请求地址、以及请求协议的版本号

②请求头：

​ 请求头有多个，每一行代表一个，请求头的作用是通知服务端的信息

​ 冒号之前的为请求头的名称，冒号之后的为请求头的值

③空白行：

​ 空白行为一段空格 ，用于分割请求头与请求体

④请求体：

​ 请求体为请求过程中所携带的额外的参数，针对post、put使用

5.3常见的请求头

  • Accept：客户端支持的格式，例如text/html
  • Accept-Encoding：客户端可以支持的压缩格式（压缩为项目优化的常见策略，例如100k的html数据压缩为10k传输个客户端，客户端解压还原为100k的文件，减少网络的传输）
  • Accept-Language：可以支持的语言
  • Referer：请求的来源。可以使用referer实现防盗链技术
  • User-Agent：客户端得到的标识信息
  • Host：请求的域名
  • Cache-Control：控制缓存的类型，常见值有no-cache（表示不缓存）max-age=0（永久不失效）
  • Cookie：cookie的标识、保存cookie信息
  • Connection：keep-alive保持长连接
  • Content-Length：在post请求中才会出现请求体的长度
  • If-Modified-Since：服务端文件的最后修改时间
  • If-None-Match：缓存标识
  • Content-Type：设置请求提交的数据的格式，常见的格式有application/x-www-form-urlencoded(URL，编码的格式，每个值使用&分割并且中文被转码)、application/json（指定json格式提交数据）、multipart/form-data（按照二进制方式传输数据即文件上传）

5.4缓存的流程

  1. 请求某一个图片地址
  2. 服务端发送响应信息。响应体为文本内容，在响应头中会存在last-modified响应头记录服务端该文件的最后修改时间。也会响应一个express响应头告诉客户端缓存时间。
  3. 客户端解析图片内容显示。当客户端可以使用缓存功能时，客户端浏览器会将图片的内容存储在计算机的某一个文件中
  4. 当下次用户访问该图片是，请求信息直接携带if-modified-since，此请求头用于该请求头记录了文件最后的修改时间。服务器收到请求时，对比服务端文件的修改时间，发现一致，直接响应使用缓存。因此减少了网路的传输。

准备工作

安装要求

保证自己电脑安装以下工具：

  • git (直接可以默认下一步)
  • nodejs(直接可以默认下一步)

安装步骤

官网地址： https://hexo.io/zh-cn/

文档：https://hexo.io/zh-cn/docs/

hexo模板主题：https://hexo.io/themes/

安装完nodejs后自带了npm工具，它是用来进行包管理的以一种工具，也可以说是用来下载更新各种库（包/框架）的工具

格式如下：

npm install xxx -g

它默认国外的网址下载(–registry=https://registry.npmjs.org)

为了下载更加快，我们通常是临时切换国内的淘宝源镜像地址下载

安装包的时候临时切换淘宝源,安装的时候加上

--registry=https://registry.npm.taobao.org

打开hexo官网

官网地址： https://hexo.io/zh-cn/

文档：https://hexo.io/zh-cn/docs/

第一步：安装 hexo-cli 脚手架工具 

npm install -g hexo-cli  --registry=https://registry.npm.taobao.org

安装本地hexo(旧版本)

npm install hexo --registry https://registry.npm.taobao.org

第二步：创建项目文件夹(进入自己某个盘，然后创建创建一个目录)

在那个目录点击shift+鼠标右键使用我们的工具执行命令

第三步:本地制作博客

  1. 初始化blog文件夹 (进入项目目录内)

    hexo init

    2.安装项目所需要的依赖

    npm install –registry=https://registry.npm.taobao.org

3.安装成功后如下:

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息,包括版本,各种依赖包信息等等

scaffolds

模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹。Hexo 会根据主题来生成静态页面。

网站(其他参数查看文档便可)

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 您的名字
language 网站使用的语言
timezone 网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

开始动工:

hexo n ‘文章的名称’

hexo g ： 生成静态页面

hexo s ： 启动本地服务(部署网站)

本地查看我们的博客效果:

http://localhost:4000

第四步:GitHub新建一个空白的博客仓库(存代码的服务器)

发布博客到GitHub,但是我们还没搭建好本地跟GitHub的关联

1.注册了一个GitHub账号

1-1 点击创建版本库

1-2 格式如下 版本库.github.io

注意:Owner跟Repository name的上马赛克部分要一致

2.打通本地和线上GitHub环境

2-1 点击设置

2-2 配置ssh

2-3 复制我们的私钥到这里

3.私钥哪里来?

3-1 打开本地gitbash

3-2 设置用户名和邮箱

# 设置你的用户名
git config --global user.name "hello"  
# 设置你的邮箱
git config --global user.email "hello@gmail.com"

3-3 生成新的SSH Key(一路确认)

ssh-keygen -t rsa -C "换成你的邮箱"

然后你可以去开始，打开你的用户目录，有一个“.ssh”文件夹，里面有 “id_rsa” “id_rsa.pub” 和 “known_hosts”这三个文件。

公钥:id_rsa.pub 在本地相当于锁头

密钥:id_rsa 放到github的ssh上,相当于钥匙

所以要把私钥(钥匙)放到github才能链接我们本地的公钥(打开锁头)

复制私钥到github

第五步:发布到线上仓库

hexo d ：发布到线上仓库 username.github.io

需要配置线上仓库：

1.配置 _config.yml 

deploy:
  type: git
  repo: git@github.com:luojiehui17/luojiehui17.github.io.git
  branch: master

  1. 安装一个工具,用与上传本地hexo到GitHub用的

    npm install hexo-deployer-git –registry https://registry.npm.taobao.org –save

hexo d
