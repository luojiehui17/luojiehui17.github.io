准备工作

安装要求

保证自己电脑安装以下工具：

  • git (直接可以默认下一步)
  • nodejs(直接可以默认下一步)

安装步骤

官网地址： https://hexo.io/zh-cn/

文档：https://hexo.io/zh-cn/docs/

hexo模板主题：https://hexo.io/themes/

安装完nodejs后自带了npm工具，它是用来进行包管理的以一种工具，也可以说是用来下载更新各种库（包/框架）的工具

格式如下：

npm install xxx -g

它默认国外的网址下载(–registry=https://registry.npmjs.org)

为了下载更加快，我们通常是临时切换国内的淘宝源镜像地址下载

安装包的时候临时切换淘宝源,安装的时候加上

--registry=https://registry.npm.taobao.org

打开hexo官网

官网地址： https://hexo.io/zh-cn/

文档：https://hexo.io/zh-cn/docs/

第一步：安装 hexo-cli 脚手架工具 

npm install -g hexo-cli  --registry=https://registry.npm.taobao.org

安装本地hexo(旧版本)

npm install hexo --registry https://registry.npm.taobao.org

第二步：创建项目文件夹(进入自己某个盘，然后创建创建一个目录)

在那个目录点击shift+鼠标右键使用我们的工具执行命令

第三步:本地制作博客

  1. 初始化blog文件夹 (进入项目目录内)

    hexo init

    2.安装项目所需要的依赖

    npm install –registry=https://registry.npm.taobao.org

3.安装成功后如下:

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息,包括版本,各种依赖包信息等等

scaffolds

模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹。Hexo 会根据主题来生成静态页面。

网站(其他参数查看文档便可)

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 您的名字
language 网站使用的语言
timezone 网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

开始动工:

hexo n ‘文章的名称’

hexo g ： 生成静态页面

hexo s ： 启动本地服务(部署网站)

本地查看我们的博客效果:

http://localhost:4000

第四步:GitHub新建一个空白的博客仓库(存代码的服务器)

发布博客到GitHub,但是我们还没搭建好本地跟GitHub的关联

1.注册了一个GitHub账号

1-1 点击创建版本库

1-2 格式如下 版本库.github.io

注意:Owner跟Repository name的上马赛克部分要一致

2.打通本地和线上GitHub环境

2-1 点击设置

2-2 配置ssh

2-3 复制我们的私钥到这里

3.私钥哪里来?

3-1 打开本地gitbash

3-2 设置用户名和邮箱

# 设置你的用户名
git config --global user.name "hello"  
# 设置你的邮箱
git config --global user.email "hello@gmail.com"

3-3 生成新的SSH Key(一路确认)

ssh-keygen -t rsa -C "换成你的邮箱"

然后你可以去开始，打开你的用户目录，有一个“.ssh”文件夹，里面有 “id_rsa” “id_rsa.pub” 和 “known_hosts”这三个文件。

公钥:id_rsa.pub 在本地相当于锁头

密钥:id_rsa 放到github的ssh上,相当于钥匙

所以要把私钥(钥匙)放到github才能链接我们本地的公钥(打开锁头)

复制私钥到github

第五步:发布到线上仓库

hexo d ：发布到线上仓库 username.github.io

需要配置线上仓库：

1.配置 _config.yml 

deploy:
  type: git
  repo: git@github.com:luojiehui17/luojiehui17.github.io.git
  branch: master

  1. 安装一个工具,用与上传本地hexo到GitHub用的

    npm install hexo-deployer-git –registry https://registry.npm.taobao.org –save

hexo d
